# Сравнительный анализ

> Сравнение пакетов локализации для Laravel, март 2026.

## Подходы к переводу моделей

| Подход | Хранение | Пакет | Скачиваний |
|--------|----------|-------|-----------|
| **JSON-колонка** | JSON в таблице самой модели | spatie/laravel-translatable | ~22.8M |
| **Отдельная таблица на модель** | `{model}_translations` таблица | astrotomic/laravel-translatable | ~7.6M |
| **Единая таблица переводов** | Одна таблица `translations`, полиморфная | **dskripchenko/laravel-translatable** | -- |

Также существуют пакеты только для **UI-строк** (не моделей): spatie/laravel-translation-loader (~2.8M), joedixon/laravel-translation (~500K).

## Детальное сравнение

### spatie/laravel-translatable

**Скачиваний:** ~22.8M | **Stars:** 2426 | **Laravel:** 11-13

Хранит переводы как JSON в колонке таблицы модели. Простейшая настройка -- один trait, одна JSON-колонка, без дополнительных таблиц.

**Сильные стороны:** нулевые накладные расходы на запросы (переводы загружаются вместе с моделью), scopes `whereLocale()` / `whereJsonContainsLocale()`, огромное сообщество.

**Ограничения:** JSON не эффективно индексируется для полнотекстового поиска, нет нормализации (все языки в одном поле), нет централизованного управления переводами, нет кэширования, каждый запрос модели загружает все языки.

### astrotomic/laravel-translatable

**Скачиваний:** ~7.6M | **Stars:** 1392 | **Laravel:** 9-12

Создаёт отдельную таблицу `{model}_translations` для каждой переводимой модели. Каждое переведённое поле получает собственную типизированную колонку.

**Сильные стороны:** полная нормализация со стандартными индексами, типизированные колонки (string, text и т.д.), fallback-локаль, JOIN-загрузка через `withTranslation()`.

**Ограничения:** взрыв таблиц (каждая модель = +1 таблица + 1 PHP-класс), дополнительные миграции при каждом изменении полей, нет встроенного кэширования, более сложная начальная настройка, менее активная поддержка (форк от заброшенного dimsav).

### lexi-translate

**Скачиваний:** <10K | **Stars:** ~30 | **Laravel:** 10-11

Использует единую таблицу `translations` с morph-связями -- архитектурно аналогичен dskripchenko.

**Сильные стороны:** одна таблица, стандартный morph-паттерн, встроенное кэширование, fallback-локаль.

**Ограничения:** молодой проект, не поддерживает Laravel 12+, ограниченная документация.

### spatie/laravel-translation-loader

**Скачиваний:** ~2.8M | **Stars:** 835 | **Laravel:** 6-13

Заменяет файловый загрузчик переводов Laravel на базу данных. Работает с `__()`, `trans()`, `@lang()` без изменений кода.

**Сильные стороны:** полная совместимость с хелперами Laravel, БД перезаписывает файлы, расширяемость (YAML/CSV провайдеры).

**Ограничения:** только UI-строки -- без переводов моделей, без управления языками.

### joedixon/laravel-translation

**Скачиваний:** ~500K | **Stars:** ~600

Полная система управления переводами с веб-интерфейсом, сканером и database-драйвером.

**Сильные стороны:** встроенный веб-редактор, сканер отсутствующих переводов, file + database драйверы, artisan-команды.

**Ограничения:** только UI-строки, менее активная поддержка, тяжелее по зависимостям.

## Когда выбирать какой пакет

| Ситуация | Рекомендация |
|----------|-------------|
| Маленький проект, 2-3 переводимые модели, минимум настройки | **spatie/laravel-translatable** -- JSON-колонка с нулевыми накладными расходами. Простейший подход, который хорошо работает при малом объёме переводов без необходимости централизованного управления. |
| Крупный проект со сложными запросами по переведённым полям, строгая типизация | **astrotomic/laravel-translatable** -- нормализованные таблицы дают полную мощь SQL с правильными типами. |
| Полный стек локализации: модели + UI + CMS + управление языками + кэширование | **dskripchenko/laravel-translatable** -- один пакет покрывает все слои. Особенно силён при необходимости автосоздания, контент-блоков и нежелании интегрировать несколько пакетов. |
| Только UI-строки из БД, существующая кодовая база с вызовами `__()` | **spatie/laravel-translation-loader** -- прозрачная замена файловых переводов без изменений кода. |
| Нужен визуальный редактор переводов для нетехнической команды | **joedixon/laravel-translation** -- поставляется с встроенным веб-интерфейсом. |
| Комбинирование пакетов | Можно использовать **dskripchenko** совместно с **spatie/translatable**, если некоторым моделям подходит JSON-хранение (например, простые конфиг-подобные поля), а другим нужна централизованная таблица переводов. Пакеты не конфликтуют. |
